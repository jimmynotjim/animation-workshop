It's really easy to abuse your animations

  • motion sickness
  • time delays
  • overly complex

Visuals shouldn't contridict your animations

  • don't use a down icon for a pop-in
  • don't use a left arrow if the interface moves right

Your animations shouldn't be lazy

  • don't use linear timing
  • do use easings
  • use delays to break up elements movement

CSS Transitions

Inform the browser how an element changes state

  • Property: background, size, location, etc
  • Duration: number in seconds or miliseconds
  • Timing Function: either pre-formatted easings or a bezier curve

.btn {
  transition-property: background;
  transition-duration: 0.25s;
  transition-timing-function: ease-out;
  transition-delay: 0.1s;
}

CSS Transitions:Shorthand

.btn {
          transition: background 0.25s;
          background-color: @pacific;

          &:hover {
            background-color: @teal;
          }
        }

CSS Transitions: Pitfalls

  • Not every property is animatable
  • Not every value is animatable
  • Some properties and value changes can be delayed with an animation despite not being animatable

CSS Transitions: Height

.list {
          transition: height 0.25s ease-out;
          height: 0;

          &:hover {
            height: auto;
          }
        }
