Properties of Good Animations

  • invisible to the user
  • responsive as we are
  • provide feedback to user input
  • inform what's happening before, during, and after a state change
  • soften out the rough spots in-between

Things are not as they seem

  • state changes in real world objects appear instant
  • there's always a transition between two states
  • nearly ever action has some kind of reaction
  • combined, these provide feedback that help guide us through the real world

Using CSS

Ninety percent of our animations can be accomplished with CSS

  • CSS Transitions

CSS Transitions

Inform the browser how an element changes state

  • Property: background, size, location, etc
  • Duration: number in seconds or miliseconds
  • Timing Function: either pre-formatted easings or a bezier curve
  • Delay: how long the browser should wait to start the transition

CSS Transitions: Longhand

.btn {
  transition-property: background;
  transition-duration: 0.25s;
  transition-timing-function: ease-out;
  transition-delay: 0.1s;
}

CSS Transitions: Delay

Delays are useful for secondary effects

.slide {
  transition: all 0.4s ease-out;
}
.slide li {
    transition: transform 0.5s ease-out;
}

.slide li:nth-of-type(1) {
  transition-delay: 0.1s;
}

.slide li:nth-of-type(2) {
  transition-delay: 0.2s;
}
etc.

CSS Transitions: Limitations

  • states are binary
  • can only be presented once
  • not really reusable

CSS Animations

  • @keyframes: define a reusable animation sequence
  • Duration: number in seconds or miliseconds
  • Timing Function: either pre-formatted easings or a bezier curve
  • Delay: how long the browser should wait to start the transition

CSS Animations: Keyframes

@keyframe rotate {
  0% {
    transform: translateZ(0deg);
  }

  100% {
    transform: translateZ(360deg);
  }
}
